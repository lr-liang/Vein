# Vein: Smart Context & Developer Navigation Hub

Vein (意为“脉络”) 是一个专为全栈开发者设计的智能化上下文导航系统。它不仅是一个前端导航页面，更是开发者在处理复杂业务逻辑与多技术栈切换时的“研发地图”。

## 🚀 项目愿景
在 AI 驱动开发的时代，开发者频繁在 **Java 后端 (Spring Cloud)**、**前端 (Angular/Vue)** 与 **AI Terminal Agents** 之间切换。Vein 旨在解决“研发上下文断层”问题，作为 **Model Context Protocol (MCP)** 的潜在接入点，为 AI 提供实时的工程环境索引。

## ✨ 核心特性 (Roadmap)
*   **智能化语义搜索**：集成 LLM 对技术文档与本地服务（如 Nacos, Spring Cloud Dashboard）进行语义化索引。
*   **研发上下文感知**：根据当前的 Git 分支或开发环境（如 Windows PowerShell 环境），动态调整工具链入口。
*   **Agent-Ready Indexing**：为 AI Agent 提供标准化的 JSON 结构化数据，使其能快速理解当前工程的微服务拓扑。
*   **现代化技术栈**：基于 pnpm 管理，采用高效的前端构建方案，支持 GitHub Pages 自动化部署。

## 🛠️ 技术实现
*   **Backend Strategy**: 深度整合 Java 生态，优化 Spring Cloud 与 Nacos 配置管理流。
*   **Frontend**: 响应式设计，适配高性能开发环境需求。
*   **Deployment**: 基于 GitHub Actions 的持续集成与发布。
