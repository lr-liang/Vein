# 菜单导入功能使用指南

## 功能概述

Vein 导航系统现在支持导入自定义菜单文件，允许用户个性化定制导航内容。

## 使用方法

### 1. 准备菜单文件

创建一个 `.yaml` 或 `.yml` 格式的菜单文件，格式如下：

```yaml
categories:
  - name: "分类名称"
    id: "category_id"
    groups:
      - name: "分组名称"
        links:
          - title: "链接标题"
            url: "https://example.com"
            description: "链接描述"
          - title: "另一个链接"
            url: "https://google.com"
            description: "Google搜索"

  - name: "另一个分类"
    id: "another_category"
    groups:
      - name: "开发工具"
        links:
          - title: "Vue.js"
            url: "https://vuejs.org"
            description: "渐进式JavaScript框架"
```

### 2. 导入菜单文件

1. 点击右上角的设置按钮
2. 在设置面板中找到"菜单管理"部分
3. 点击"导入菜单文件"按钮
4. 选择你准备好的 YAML 文件

### 3. 验证导入结果

- 导入成功后会显示成功提示
- 页面会自动刷新并应用新的菜单
- 设置面板会显示"已导入自定义菜单"状态

### 4. 恢复默认菜单

如果需要恢复到默认菜单：

1. 在设置面板的"菜单管理"部分
2. 点击垃圾桶图标按钮
3. 系统会清除自定义菜单并使用默认的 `menu.yaml`

## 文件格式要求

### 必需字段

- `categories`: 分类数组（必需）
- 每个分类必须有：
  - `name`: 分类名称
  - `id`: 分类唯一标识符
  - `groups`: 分组数组
- 每个分组必须有：
  - `name`: 分组名称
  - `links`: 链接数组
- 每个链接必须有：
  - `title`: 链接标题
  - `url`: 链接地址

### 可选字段

- `description`: 链接描述（显示在悬停时）

## 数据存储

- 导入的菜单文件会保存在浏览器的 IndexedDB 中
- 刷新页面或重新访问时，系统会优先加载已保存的自定义菜单
- 清除浏览器数据会同时清除保存的菜单

## 错误处理

系统会验证文件格式并提供友好的错误提示：

- ❌ 文件格式不正确（非YAML格式）
- ❌ 缺少必需字段
- ❌ 文件结构不符合要求
- ✅ 导入成功提示

## 注意事项

1. 多次导入会完全替换之前的自定义菜单
2. 菜单数据仅在当前浏览器中有效
3. 建议备份重要的菜单配置文件
4. 支持标准的 YAML 语法，包括注释等
